# Playbook

## Sources
- Design for Cognitive Bias — David Dylan Thomas
- Laws of UX — Jon Yablonski

## Objetivo
Usar psicologia para orientar e simplificar — nunca para manipular e prender.

## Regra Central
Design responsável aproveita vieses para simplificar.
Dark pattern explora vieses para manipular.

## Vieses a Vigiar
- Framing Effect: enquadramento informa ou distorce?
- Loss Aversion: cópia alarmista para forçar ação?
- Scarcity: escassez informativa ou fabricada?
- Cognitive Fluency: facilidade visual não valida claims falsos.
- Peak-End: erro crítico no final destrói memória positiva acumulada.

## Positive Friction — Quando Usar
Decisão irreversível, financeiramente sensível, de privacidade, juridicamente relevante.
Inclui: confirmações claras, resumo da consequência, opção de desfazer.

## Dark Pattern Red Flags
- Botão principal ambíguo ou enganoso.
- Cancelamento oculto sem necessidade.
- Opt-in pré-marcado para cobrança recorrente.
- Urgência falsa sem base real.
- Bloqueio artificial para exportar/apagar dados.

## Review Protocol
1. O usuário entende o que está consentindo?
2. Existe caminho para recusar sem punição?
3. Consequências explicadas em linguagem simples?
4. Design protege usuários vulneráveis?
