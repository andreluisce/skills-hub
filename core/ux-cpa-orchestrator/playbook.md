# Playbook

## Objetivo
Coordenar análise completa de UX integrando todas as skills especializadas do pacote.

## Quality Bar
A recomendação final só é válida quando:
1. Reduz carga cognitiva.
2. Melhora resultado mensurável.
3. Não usa manipulação antiusuário.
4. É viável para implementar.
5. Melhora ou preserva performance.

## Orchestration
1. ux-cognitive-architecture — custo mental, ambiguidades, excesso de escolha.
2. ux-behavioral-loop — trigger, ação, recompensa, investimento.
3. ux-visual-systems — hierarquia, escala, espaçamento, tokens.
4. ux-lean-experimentation — hipóteses testáveis com métricas.
5. ux-ethics-bias — dark patterns, atrito positivo.
6. ux-sustainable-product — performance e eficiência técnica.
7. ux-emotional-stakeholder — proposta defensável para negócio e engenharia.

## Output Structure
1. Problema sintetizado.
2. Diagnóstico por camada.
3. Plano priorizado P0/P1/P2.
4. Hipóteses e métricas.
5. Riscos éticos e mitigações.
6. Impacto técnico/performance.
7. Script de comunicação para stakeholders.
