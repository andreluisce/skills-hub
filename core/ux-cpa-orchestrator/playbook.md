# Playbook

## Objective
Coordinate complete UX analysis integrating all specialized skills in the package.

## Quality Bar
The final recommendation is only valid when:
1. Reduz carga cognitiva.
2. Measurable improvement in results.
3. Does not use antiuser handling.
4. It is feasible to implement.
5. Melhora or preserva performance.

## Orchestration
1. ux-cognitive-architecture — custo mental, ambiguidades, excesso of escolha.
2. ux-behavioral-loop — trigger, action, reward, investment.
3. ux-visual-systems — hierarchy, scale, spacing, tokens.
4. ux-lean-experimentation — hypothesiss testable with metrics.
5. ux-ethics-bias — dark patterns, atrito positivo.
6. ux-sustainable-product — performance and technical efficiency.
7. ux-emotional-stakeholder — defensible proposal for business and engineering.

## Output Structure
1. Problema sintetizado.
2. Diagnosis by layer.
3. Plano priorizado P0/P1/P2.
4. Hypotheses and metrics.
5. Ethical risks and mitigations.
6. Technical impact/performance.
7. Script of communication for stakeholders.
