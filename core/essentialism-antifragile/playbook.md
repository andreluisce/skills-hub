# Playbook

## Sources
- Essencialismo — Greg McKeown
- Antifragil — Nassim Nicholas Taleb
- A Revolução 80/20 — Richard Koch

## Objetivo
Fazer apenas o essencial e construir sistemas que melhoram sob pressão.

## Regras de Essentialism
- Se não é um "sim óbvio", é um não.
- Código não escrito não tem bugs, não precisa de manutenção.
- Cada abstração justifica sua existência com uso real, não potencial.
- Não construa para hipotéticos — construa para o que existe hoje.

## Regras de Antifragile
- Deploys frequentes e pequenos > deploys grandes e raros.
- Feature flags: rollback imediato = sistema antifragil.
- Skin in the game: quem projeta também faz on-call.
- Optionality: evite lock-in sem escape hatch.

## 80/20
- 20% de features geram 80% do uso — invista ali.
- 20% de arquivos têm 80% dos bugs — refatore ali primeiro.
- 20% de fluxos são 80% do uso — otimize performance ali.
