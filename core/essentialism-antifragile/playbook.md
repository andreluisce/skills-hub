# Playbook

## Sources
- Essencialismo — Greg McKeown
- Antifragil — Nassim Nicholas Taleb
- A Revolution 80/20 — Richard Koch

## Objective
Do only the essentials and build systems that improve under pressure.

## Rules of Essentialism
- If not is an "obvious yes", it is a not.
- Code not written does not have bugs, does not need maintenance.
- Each abstraction justifies its existence with actual use, not potential.
- Don't build on hypotheticals — build on what exists today.

## Rules of Antifragile
- Deploys frequentes e pequenos > deploys grandes e raros.
- Feature flags: rollback imediato = sistema antifragil.
- Skin in the game: whoever designs also makes on-call.
- Optionality: evite lock-in without escape hatch.

## 80/20
- 20% of features geram 80% do uso — invista ali.
- 20% of files have 80% of the bugs — refactor there first.
- 20% of flows are 80% of usage — optimize performance there.
