# UX Ethics and Cognitive Bias

## Purpose
Esta skill existe para impedir que principios de psicologia sejam usados apenas para elevar conversao de curto prazo. O objetivo e aumentar qualidade de decisao do usuario, confianca no produto e longevidade da relacao.

## Core Position
Vieses cognitivos sao inevitaveis no comportamento humano. O design pode:

1. aproveitar vieses para simplificar e orientar,
2. explorar vieses para manipular e prender.

A primeira opcao e design responsavel. A segunda e dark pattern.

## Biases to Consider in Product Decisions

### Framing effect
A forma da mensagem altera a decisao. Sempre validar se o enquadramento informa ou distorce.

### Loss aversion
Perda pesa mais que ganho. Evitar copia alarmista para forcar compra/acao.

### Scarcity
Escassez pode informar contexto real (estoque, prazo legal), mas nunca deve ser fabricada.

### Cognitive fluency
Facilidade visual aumenta percepcao de verdade. Exigir rigor adicional em claims de marketing e mensagens de risco.

### Negativity bias + Peak-End
Um unico erro critico no fim da jornada pode destruir memoria positiva acumulada. Fluxos finais devem priorizar clareza, seguranca e confirmacao.

## Positive Friction Rules
Use atrito deliberado quando a decisao e:

1. irreversivel,
2. financeiramente sensivel,
3. de privacidade/dado pessoal,
4. juridicamente relevante.

Atrito positivo inclui confirmacoes claras, resumo da consequencia, e opcao de desfazer quando viavel.

## Dark Pattern Red Flags

1. Botao principal ambiguo ou enganoso.
2. Cancelamento oculto em submenus sem necessidade.
3. Opt-in pre-marcado para cobranca recorrente.
4. Copia de urgencia falsa ("ultimas vagas" sem base real).
5. Bloqueio artificial para exportar/apagar dados.

## Review Protocol
Antes de aprovar uma mudanca, responder:

1. O usuario entende o que esta consentindo?
2. Existe caminho claro para recusar sem punicao indevida?
3. A interface informa consequencias com linguagem simples?
4. O design protege usuarios vulneraveis em cenarios de pressa/fadiga?

## Deliverables

1. Auditoria de vieses por fluxo.
2. Matriz de risco etico (baixo/medio/alto impacto).
3. Plano de friccao positiva para acoes criticas.
4. Lista de ajustes de copy para consentimento explicito.
