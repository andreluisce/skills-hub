# UX Behavioral Loop

## Purpose
Esta skill desenha comportamento recorrente sem cair em manipulacao barata. O objetivo e criar retorno voluntario porque o produto resolve dor real de forma repetida.

## Behavioral Model
Use o ciclo:

1. Trigger
2. Action
3. Variable Reward
4. Investment

Com validacao continua para garantir que o loop aumenta valor para o usuario e nao so dependencia.

## Operational Principles

### 1) Trigger design
Iniciar por gatilhos externos (notificacao, email, agenda), evoluindo para gatilhos internos legitimos (ex.: necessidade de acompanhar status, evitar retrabalho).

### 2) Action simplification (Fogg)
Comportamento exige motivacao + habilidade + gatilho. A habilidade (facilidade) costuma ter maior ROI: remover passos, clarificar acao, reduzir friccao motora e cognitiva.

### 3) Variable reward com criterio
Recompensa variavel pode ser social, informacional ou de progresso. Deve reforcar valor funcional, nao gerar ansiedade artificial.

### 4) Investment
Pedir pequenas contribuicoes que aumentam utilidade futura (preferencias, dados, colaboracao), preparando o proximo retorno.

## Decision Guardrails

1. O loop reduz custo de vida/trabalho do usuario?
2. O usuario tem controle sobre notificacoes e frequencia?
3. O engajamento vem de utilidade ou medo de perder algo?
4. Existe transparencia sobre o que e opcional vs obrigatorio?

## Deliverables

1. Mapa de loop comportamental por etapa.
2. Tabela de friccao (onde reduzir, onde manter, onde aumentar atrito positivo).
3. Plano de experimento de retencao com metricas D1/D7/D30.

## Anti-Patterns

1. Escassez falsa.
2. Urgencia artificial constante.
3. Notificacao sem valor contextual.
4. Retencao sem ganho de resultado para o usuario.
