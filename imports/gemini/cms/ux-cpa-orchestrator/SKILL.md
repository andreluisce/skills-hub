# UX CPA Orchestrator

## Purpose
Esta skill funciona como camada de coordenacao para problemas complexos de produto. Em vez de respostas isoladas, ela monta um plano integrado usando as skills especializadas do pacote UX.

## Inputs Expected

1. Contexto do produto e meta de negocio.
2. Fluxo ou tela alvo.
3. Sintoma principal (queda de conversao, abandono, erro, baixa retencao, baixa confianca).
4. Restricoes tecnicas/operacionais.

## Orchestration Flow

### Step 1 - Cognitive diagnosis
Aplicar `ux-cognitive-architecture` para identificar custo mental, ambiguidades e excesso de escolha.

### Step 2 - Behavior loop mapping
Aplicar `ux-behavioral-loop` para mapear trigger, acao, recompensa e investimento.

### Step 3 - Visual system correction
Aplicar `ux-visual-systems` para hierarquia, escala, espacamento, contraste e tokens.

### Step 4 - Product experiment plan
Aplicar `ux-lean-experimentation` para transformar mudancas em hipoteses testaveis com metricas.

### Step 5 - Ethical gate
Aplicar `ux-ethics-bias` para detectar risco de dark pattern e definir atrito positivo.

### Step 6 - Sustainability gate
Aplicar `ux-sustainable-product` para garantir eficiencia tecnica e ambiental.

### Step 7 - Stakeholder narrative
Aplicar `ux-emotional-stakeholder` para gerar proposta defensavel por negocio, produto e engenharia.

## Required Output Structure

1. Problema sintetizado.
2. Diagnostico por camada (cognitiva, visual, comportamental).
3. Plano de mudanca priorizado (P0/P1/P2).
4. Hipoteses e metricas de validacao.
5. Riscos eticos e mitigacoes.
6. Impacto tecnico/performance.
7. Script de comunicacao para stakeholders.

## Quality Bar
A recomendacao final so e valida quando:

1. reduz carga cognitiva,
2. melhora resultado mensuravel,
3. nao usa manipulacao antiusuario,
4. e viavel para implementar,
5. melhora ou preserva performance.
