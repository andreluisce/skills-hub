# Skills Hub

Centralized multi-agent skill management system. Packages domain knowledge as structured skills and transpiles them to Claude, Codex, and Gemini formats. Follows a **catenary architecture** with strict separation of concerns.

## Architecture

```
core/          → 57+ canonical skills (skill.yaml + playbook.md)
agents/        → Role-based agent contracts (.agent.yaml)
adapters/      → Transpilers: core skills → dist/{claude,codex,gemini}
context/       → This file + adapter that generates *MD context files
cli/           → skills-hub.mjs: init | validate | build | context | audit
schemas/       → skill.schema.json (JSON Schema v7, validates all skills)
mcp/registry/  → mcp-registry.yaml (MCP server declarations)
dist/          → Generated artifacts (do not edit manually)
references/    → LLM_SYSTEM_BLUEPRINT.md, ROADMAP.md
squads/        → 12 cross-disciplinary AI agent squads (xquads)
```

## Key commands

```bash
# Toolkit development
pnpm validate        # Validate all skills against skill.schema.json
pnpm build           # Transpile all skills to dist/{claude,codex,gemini}
pnpm run context     # Regenerate CLAUDE.md, AGENTS.md, GEMINI.md (toolkit self-docs)
pnpm test            # Smoke test: validate + build

# Install into a target app repo
npx skills-hub init  # Interactive install — generates all context, skills, agents, MCP
```

## What `npx skills-hub init` generates in a target app

```
CLAUDE.md / AGENTS.md / GEMINI.md           context for all 3 AI tools
.agents/skills/<slug>/SKILL.md              Open Agent Skills Standard (Codex + Gemini)
.agents/skills/<slug>/agents/openai.yaml    Codex UI metadata
.claude/skills/<slug>/SKILL.md              Claude skill modules with frontmatter
.claude/commands/<slug>.md                  slash commands (/slug)
.claude/commands/plan.md                    cross-model plan workflow (/plan)
.claude/agents/<agentId>.md                 agents with model/maxTurns/memory/skills frontmatter
.claude/agent-memory/<agentId>/MEMORY.md    persistent agent memory stubs
.mcp.json                                   project MCP servers (context7 + playwright + deepwiki + stack-specific)
.claude/settings.json                       Claude permissions + autoCompact@80% + MCP
.gemini/settings.json                       Gemini context file config
```

## Orchestration pattern (Command → Agent → Skill)

```
/plan or /<skill>   →   .claude/commands/      entry point, user interaction
                    →   .claude/agents/        executes with preloaded skills, model, memory
                    →   .claude/skills/        knowledge module (preloaded or invoked)
```

## Open Agent Skills Standard

All three tools converge on the same standard at `.agents/skills/`:
- **Claude**: also reads `.claude/skills/`
- **Codex**: reads `.agents/skills/` (walks up from CWD)
- **Gemini**: reads `.gemini/skills/` or `.agents/skills/`

## Conventions

- Skill names: `kebab-case`, 3–64 chars, unique
- Each skill: `core/<slug>/skill.yaml` + `core/<slug>/playbook.md`
- All skills must pass `pnpm validate` before committing
- `dist/` is fully regenerated by `pnpm build` — never edit manually
- MCP servers: register in `mcp/registry/mcp-registry.yaml`
- Agent contracts: `agents/<role>.agent.yaml`
- Use `pnpm run context` after editing this file

## Skill schema (required fields)

`schemaVersion`, `name`, `displayName`, `description`, `triggers`, `stack`, `workflow` (≥3 steps), `outputContract` (≥1)

## Stack defaults (most skills target)

- Framework: Next.js 15+ (App Router)
- React: React 19+
- Server state: TanStack Query v5
- UI: shadcn/ui + Tailwind v4
- Workspace: pnpm monorepo
